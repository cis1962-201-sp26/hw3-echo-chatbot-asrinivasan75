# Chat

# Functionality

- [ ] User is able to send messages to the chatbot
- [ ] User is able to receive messages from the chatbot
- [ ] Chatbot remembers the conversation upon reload
- [ ] Chat history is stored in localStorage

# HTML

- [ ] HTML is well structured can contains the main components required by the app, including a header, main chat area, an input field, a send button, and a new chat button
- [ ] HTML uses proper semantic tags rather than just <div> and <span> everywhere
- [ ] HTML uses meaningfully named IDs and classes for styling and DOM manipulation

# CSS

- [ ] CSS makes use of at least one flexbox, if not more to properly layout elements
- [ ] CSS makes good use of colors, either built-in or though hex or RGB codes
- [ ] App makes good use of the box model with proper margins, padding, and borders where applicable

# UI/UX

- [ ] Send button is visible and clickable
- [ ] Input field is visible and clickable
- [ ] Send button is disabled when the input field is empty
- [ ] Chat messages are displayed in a scrollable container, with consideration to both large and small screen sizes
- [ ] Sender and receiver are displayed in a different color

# Code

- [ ] script.ts is written in TypeScript, and compiles to script.js within the dist folder
- [ ] Code has no TypeScript errors, warnings, or bugs
- [ ] No 'any' types are parsed in TypeScript (unless a good reason is provided)

Note: We will only style check the .ts file!
